When testing ranges, did you use &&, ||, or something else entirely (e.g. nested if-statements)? If you used &&, why is that a better choice than || (and vice-versa)?

I used && for the if (user_age <= 15 && user_age >= 0)

Because the user's age must produce true for that if statement to happen. I couldn't really use || is this statement as both needed to be true, not one.
